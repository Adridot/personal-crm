# Execution Standards and Sources

Use these sources to guide implementation execution:

- GitHub: [GitHub flow](https://docs.github.com/en/get-started/using-github/github-flow)
- Atlassian: [Definition of Done](https://www.atlassian.com/agile/project-management/definition-of-done)
- Trunk Based Development: [Short-lived branches](https://trunkbaseddevelopment.com/short-lived-feature-branches/)

Execution quality checks:

- Keep implementation scoped to approved ticket boundaries.
- Run the smallest effective validation set, then expand if risk increases.
- Surface blockers quickly and request explicit user decisions for crucial trade-offs.
- Keep technical traceability updated in issue and docs.
