# Planning Standards and Sources

Use these sources first when building plans:

- GitHub Issues: [About issues](https://docs.github.com/en/issues/tracking-your-work-with-issues/about-issues)
- GitHub Projects: [About Projects](https://docs.github.com/en/issues/planning-and-tracking-with-projects/learning-about-projects/about-projects)
- Agile Alliance: [INVEST in user stories](https://www.agilealliance.org/glossary/invest/)
- Scrum Guides: [The Scrum Guide](https://scrumguides.org/scrum-guide)

Planning quality checks:

- Ensure the scope is independent, negotiable, and testable.
- Prefer small tickets with a clear acceptance signal.
- Explicitly separate product decisions from implementation decisions.
- Track assumptions and unresolved decisions as blockers.
